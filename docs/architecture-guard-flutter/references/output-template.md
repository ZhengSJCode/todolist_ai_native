# 架构健康度报告输出模板

严格按照以下格式输出，不增减章节。

---

```
## 🏗️ 架构健康度报告

**场景**：[TDD / 功能实现 / Code Review]
**文件/模块**：[文件名或模块名]

---

### 📊 综合健康度：X.X / 10

| 原则 | 分数 | 权重 | 加权分 |
|------|------|------|--------|
| SRP 单一职责 | X/10 | 20% | X.X |
| LSP 里氏替换 | X/10 | 15% | X.X |
| DIP 依赖注入 | X/10 | 20% | X.X |
| ISP 接口分离 | X/10 | 15% | X.X |
| LoD 迪米特   | X/10 | 15% | X.X |
| OCP 开闭原则 | X/10 | 15% | X.X |
| **综合**     |      |      | **X.X** |

---

### 🔍 问题清单

#### 🔴 Critical（必须修复）
- **[原则缩写]** 问题描述，影响说明

#### 🟡 Warning（建议修复）
- **[原则缩写]** 问题描述

#### 🔵 Suggestion（可选优化）
- **[原则缩写]** 优化建议

#### ✅ 良好设计
- 值得保持的架构决策

---

### 🔧 重构建议

> 每条建议包含：违反原则说明 + diff

#### 建议 1：[简短标题]

违反 [原则]：[一句话说明为什么要改]

```diff
- // 原代码
+ // 修改后代码
```

#### 建议 2：...（如有）

---

### 🧩 设计模式建议

[如满足引入条件，列出建议模式和理由]
[如不满足，写："当前复杂度不需要引入设计模式，保持简单实现即可。"]

---

### 📋 TDD 场景补充（仅 TDD 场景输出此节）

**接口设计建议：**
- [针对接口定义的具体建议]

**可测试性评估：**
- [当前设计是否便于 mock 和单元测试]
```
